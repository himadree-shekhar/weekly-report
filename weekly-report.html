🌐 PHPKB SIDE:

phpkb_articles
└── article_id ← Unique article
└── article_title ← Human-readable title (e.g. "Pathways Network Editor")
└── ...many fields...

phpkb_custom_fields
└── field_id
└── field_name ← (e.g. "CHM Help ID")
└── field_type ← (text/integer/etc.)

phpkb_custom_data
└── data_id
└── field_id ← FK to 'CHM Help ID' field
└── article_id ← FK to actual article
└── field_data ← Actual value (e.g., 10225)

- Add 1 custom field to phpkb_custom_fields:
- Then store mappings in phpkb_custom_data: Each row:
    field_id = ID of “CHM_HELP_ID”
    article_id = PHPKB article (matching the help topic)
    field_data = CHM ID (e.g., 10225)

#define TopicName 10225 ← From CHM .hh file

-- This query finds the article in PHPKB that has a custom field "CHM_HELP_ID" with the value 10225
SELECT
a.article_id,
a.article_title
FROM
phpkb_articles a
JOIN phpkb_custom_data d ON a.article_id = d.article_id -- Link custom data to article
JOIN phpkb_custom_fields f ON d.field_id = f.field_id -- Link custom data to custom field
WHERE
f.field_name = 'CHM_HELP_ID' -- Filter to only use CHM Help ID field
AND d.field_data = '10225'; -- Lookup CHM Help ID (from Delphi)
